# Mitsukami2YokohamaViaBus.js ドキュメント

## 概要

このモジュールは、**「国大周辺（現在地） → バス → 三ッ沢上町駅 → 電車（ブルーライン） → 横浜駅」** というルートの経路探索・結合ロジックを担当します。

複数のバス事業者（相鉄、神奈中、市営）の時刻表データを取得・統合し、三ッ沢上町駅での電車乗り換えを考慮した最適なルートリストを生成して返します。

文責：青柳,Gemini

## 依存関係

このモジュールは以下の内部モジュールに依存しています。

* `../utils/utils.js`: 時刻計算などのユーティリティ
* `./kaikan2Yokohama.js`: 相鉄バス データ取得用
* `./kanachu2Yokohama.js`: 神奈中バス データ取得用
* `./YokohamaMunicipalBus2Yokohama.js`: 横浜市営バス データ取得用
* `./Mitsukami2Yokohama.js`: 三ッ沢上町駅発の電車データ取得用

## 主な処理フロー

`Mitsukami2YokohamaViaBus` 関数がメインのエントリーポイントです。

1. **バスデータ取得 (`modifyAllBusData`)**
* 3つのバス事業者API/スクレイパーを並列実行 (`Promise.allSettled`) します。一部が失敗しても処理を継続します。
* 取得したデータをフラット化し、**三ッ沢上町への到着時刻順**にソートします。


2. **到達可能性判定 (`checkReachability`)**
* ユーザーの現在地から各バス停（国大西、正門、岡沢町など）への徒歩時間を考慮し、そのバスに「間に合うか」を判定します。


3. **最適バス停の選定**
* 1つのバス便に対し、複数のバス停から乗車可能な場合、以下の優先順位で乗車バス停を決定します。
1. **滞在時間最大化**: 現在地を出発する時刻が最も遅くなるバス停。
2. **徒歩最短**: 出発時刻が同じ場合、徒歩時間が短いバス停。




4. **電車データとの結合**
* バスの「三ッ沢上町」到着時刻に、乗り換えバッファ（定数 `MitsukamiBusStopToStation = 4分`）を加算した時刻以降に出発する電車を検索します。
* 最大3件の接続電車候補を抽出します。


5. **レスポンス生成**
* バスと電車をセットにしたオブジェクト配列を返却します。



## 設定・定数 (`const`)

開発時に調整が必要な主なパラメータは以下の通りです。

| 変数名 | 説明 | 現状の値 |
| --- | --- | --- |
| `busStopMap` | 内部キー（`ToKokudaiNishi`等）とAPI上のバス停名のマッピング定義 | 配列（詳細コード参照） |
| `BUS_FARES` | 事業者ごとのバス運賃（円） | 市営/神奈中: 220, 相鉄: 240 |
| `MitsukamiBusStopToStation` | バス降車から駅ホームまでの移動バッファ（分） | `4` |
| `maxCounter` | 1つのバス便に対して提示する接続電車の最大数 | `3` |

## 入出力仕様

### 入力: `paramJSON` (Object)

検索条件を含むオブジェクトを受け取ります。

```javascript
{
  "currentTime": "12:14", // 現在時刻 (HH:mm)
  "location": "Library",  // 現在地の名称（デバッグ/ログ用）
  "walkspeed": "fast",    // 徒歩速度（現状ロジックには直接影響しないが引き回される）
  "isholiday": false,     // 休日ダイヤ判定用フラグ
  // 以下、現在地から各バス停への徒歩所要時間（分）
  "ToKokudaiNishi": 10,
  "ToKokudaiChuo": 5,
  "ToKaikan": 6,
  "ToMinamiG": 7,
  "ToShindo": 8,
  "ToOkazawacho": 9
}

```

### 出力: `returnArray` (Array of Objects)

計算されたルートのリストを返します。

```javascript
[
  {
    "index": 1,
    "departuretime": "12:20",     // ユーザーが現在地を出発すべき時刻
    "arrivetime": "12:35",        // 三ッ沢上町バス停到着時刻
    "fare": 440,                  // バス + 電車の合計運賃
    "busstop_name": "国大中央",    // 乗車すべきバス停
    "busstop_walktime": 5,        // バス停までの徒歩時間
    "busstop_departuretime": "12:25", // バスの発車時刻
    "busstop_dropoff_name": "三ツ沢上町",
    "busstop_dropoff_arrivetime": "12:35",
    // 接続する電車情報（筆頭候補）
    "station_name": "三ッ沢上町",
    "station_departuretime": "12:40",
    "station_dropoff_name": "横浜駅",
    "station_dropoff_arrivetime": "12:45",
    // デバッグ・詳細表示用データ
    "busdata": { ... }, 
    "traindata": [ ... ] // 接続候補の電車リスト（最大3件）
  },
  // ... 他の候補
]

```

## 実装上の注意点・既知の課題

1. **バス停名の表記ゆれ対応**
* APIごとに「横浜新道」「横浜新道(新道)」など表記が異なるため、`busStopMap` の `stopNames` 配列で吸収しています。新しいデータソースを追加する場合はここを確認してください。


2. **乗り換え時間の固定**
* `MitsukamiBusStopToStation` は現在固定値（4分）です。個人の歩行速度（`walkspeed`）に応じて動的に変更する実装は `TODO` となっています。


3. **運賃計算**
* `getBusFare` ヘルパー関数で事業者名文字列（`operatorName`）から判定していますが、文字列マッチングがハードコード気味です。データソース側のフォーマット変更に注意してください。


4. **電車の絞り込み**
* バス到着時刻より前に出る電車は除外されます。また、接続可能な電車がない場合、そのバス便自体が結果から除外されます。
